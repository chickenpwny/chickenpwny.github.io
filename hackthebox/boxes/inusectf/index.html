    CTF

    The biggest take away would be

    I recently participated in a ctf, it was different than htb. more about finding a bunch of hidden flags all over the file system. in various places. There were 9 host and over 25 flags. This was a fun new kind of a ctf. =].DONT OVERESTIMATE THE CTF.

    WebServer: It had a lot of ports open an I was quick to rush in and wasted a bit of time that couldn’t have been prevented. Anyways I thought it was mistakenly a windows host. I should have run a targeted scan next. Just ran a normal Nmap scan instead of using the -sT switch which will perform a TCP connection to the port and it has to respond. It’s quicker. Once on the system i used sshuttle to pivot and gaina access to 172.16.42.0/24 subnet.

    Router: speculation on how to exploit. It was a old d-link router that had several vulnerabilities I couldn’t get to work. It had a buffer overflow vulnerability that the pubic poc didn’t work but im willing to bet that with modification it might have worked. Because the version was deprecated I feel like this one required custom exploitation.

    FireWall: From what I could tell this was a updated or recently updated version of pfsense. as I uses this at home. So I tried default credentials to login but couldn’t gain access. I also ran hydra against the target.

    FileServer: Had some crap on it. I would like to note the ftp server had some filters up and could only be accessed via browser.

    WebServer Dog: This seemed to be some kind of social media mortgage lender. This one I thought was interesting mostly because I thought there might be some saved credentials that might help out with eh domain controller.

    Domain Controller : I wasn’t exactly sure how the firewall and the router played into perhaps gaining access to the dc. Or perhaps I didn’t find the credentials need to gain access to the dc. I ran various tools to try and get it to dump creds & to enumerate it further. Guest access wasn’t enabled. I also ran hydra against this and tried to brute force dc creds. I also tried some kerb attacks that didn’t lead anywhere. The system clock could have been off that would be a simple easy trick to fool people someone into thinking guest isn’t enabled. On the same note the whole environment system clock could be off.

    The dotted lines represent a hypothetical connection between the firewall and dc. note that the router is 172.16.1.3

So from doing hacktheboxes, I knew I could use a for loop and nc to enumerate the other subnet without having to move over a nmap binary. Because of the names in the host list in the menu I knew one was a DC so I also used port 88 instead of 22.
